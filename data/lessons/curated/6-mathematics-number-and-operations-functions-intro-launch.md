# Functions (intro)

## Learning Goal
- I can identify the input, output, and rule in a function table.
- I can decide whether each input has exactly one output.
- I can write an equation or verbal rule for a simple function.

## Introduction
In Grade 6, a function is a rule that matches an input to an output.

You can think of a function as a machine:

- a number goes in
- the rule changes it
- a new number comes out

The most important idea is consistency. If the same input appears twice, it must always produce the same output. If one input is matched with two different outputs, the relationship is not a function.

Functions can be shown in several forms:

- input-output tables
- ordered pairs
- graphs
- equations
- verbal rules

In this lesson, focus on three questions:

1. How can I tell whether a relationship is a function?
2. How do I describe the rule from input to output?
3. How can I use the rule to predict a missing value?

## Key Vocabulary
**Function**: A relationship in which each input is paired with exactly one output.

**Input**: The starting value in a function.

**Output**: The value produced by the rule.

**Rule**: The pattern or operation that connects each input to its output.

**Ordered pair**: Two numbers written as `(x, y)` to show an input and an output.

**Table**: A chart that lists matching input and output values.

## What Makes a Relationship a Function?
A relationship is a function when each input has only one output.

Example:

- `2 -> 7`
- `3 -> 9`
- `4 -> 11`

This can be a function because each input is matched with one result.

Now look at this example:

- `5 -> 12`
- `5 -> 13`

This is not a function because the same input, `5`, has two different outputs.

That simple check is often the fastest way to decide.

Ask:

1. Does any input repeat?
2. If it repeats, does it always lead to the same output?

If the repeated input gives different outputs, the relationship fails the function rule.

## Worked Example 1: Reading a Function Table
The table shows the number of notebooks and the total cost.

| Input: notebooks | Output: cost |
| --- | --- |
| 1 | 4 |
| 2 | 7 |
| 3 | 10 |
| 4 | 13 |

Step 1: Check whether each input has one output.

Each input appears once, so the table can represent a function.

Step 2: Look for the pattern.

As the number of notebooks increases by `1`, the cost increases by `3`.

Step 3: Describe the rule.

The output is always `1` more than `3` times the input.

That gives the rule:

`output = 3 x input + 1`

If we use `x` for the input and `y` for the output, the equation is:

`y = 3x + 1`

Check with `x = 4`:

`y = 3(4) + 1 = 13`

The rule matches the table.

## Worked Example 2: Finding a Missing Output
A function follows the rule:

`y = 5x - 2`

Find the output when the input is `6`.

Step 1: Substitute `6` for `x`.

`y = 5(6) - 2`

Step 2: Multiply first.

`y = 30 - 2`

Step 3: Subtract.

`y = 28`

So when the input is `6`, the output is `28`.

This is one of the main jobs of a function rule: it lets you predict values without listing every pair in a table.

## Worked Example 3: Finding a Missing Input
A function machine adds `8` to an input.

One output is `19`. What input made that output?

Step 1: Write the rule.

`y = x + 8`

Step 2: Replace `y` with `19`.

`19 = x + 8`

Step 3: Subtract `8` from both sides.

`11 = x`

So the missing input is `11`.

Check:

`11 + 8 = 19`

The result fits the rule.

## From a Verbal Rule to Ordered Pairs
Suppose the rule is:

"Multiply the input by `2`, then add `3`."

That rule can be written as:

`y = 2x + 3`

Use a few inputs:

- if `x = 0`, then `y = 3`
- if `x = 1`, then `y = 5`
- if `x = 2`, then `y = 7`
- if `x = 5`, then `y = 13`

So some ordered pairs are:

- `(0, 3)`
- `(1, 5)`
- `(2, 7)`
- `(5, 13)`

All of those points follow the same rule, so they belong to the same function.

## Worked Example 4: Deciding Whether Ordered Pairs Form a Function
Decide whether these ordered pairs represent a function:

- `(1, 6)`
- `(2, 9)`
- `(3, 12)`
- `(2, 9)`

Step 1: Look at the inputs.

The inputs are `1`, `2`, `3`, and `2`.

Step 2: Check the repeated input.

The input `2` repeats, but it gives the same output both times: `9`.

That means the relationship is still a function.

Repeated ordered pairs do not break a function. A function only fails when one input is paired with different outputs.

## Common Mistake: Looking Only at the Outputs
Students sometimes think a relationship is not a function because two outputs are the same.

Example:

- `(1, 4)`
- `(2, 4)`
- `(3, 7)`

This is still a function.

Why?

Each input has exactly one output. It is allowed for different inputs to share the same output.

The real question is not:

"Do any outputs repeat?"

The real question is:

"Does any one input point to more than one output?"

## Summary
A function is a relationship in which each input has exactly one output.

To work with an introductory function:

1. check whether each input is matched to only one output
2. look for the rule connecting input and output
3. write the rule in words, a table, or an equation
4. use the rule to predict missing inputs or outputs

When you can move between tables, ordered pairs, and equations, functions become much easier to understand and use.
