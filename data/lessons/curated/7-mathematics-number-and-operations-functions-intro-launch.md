# Functions (intro)

## Learning Goal
- I can identify whether a relationship is a function from a table, graph, equation, or set of ordered pairs.
- I can describe how a change in the input affects the output.
- I can compare two functions using their rules, tables, or graphs.

## Introduction
In Grade 7, functions become more than simple input-output machines. You begin comparing relationships, analyzing how outputs change, and moving between several representations of the same rule.

A function still follows the same basic requirement:

- each input has exactly one output

But now the work is more flexible. You may need to:

- read a function table
- write a rule from a pattern
- compare two functions
- interpret what the numbers mean in context

In this lesson, focus on three questions:

1. How can I recognize a function in different forms?
2. How do I describe the pattern or rate of change in a function?
3. How can I compare two functions to decide which grows faster or starts higher?

## Key Vocabulary
**Function**: A relationship in which each input is paired with exactly one output.

**Input**: The independent value that goes into a function.

**Output**: The dependent value produced by the function.

**Ordered pair**: Two related numbers written as `(x, y)`.

**Rate of change**: How much the output changes when the input changes by `1`.

**Initial value**: The output when the input is `0`.

## Recognizing a Function Across Representations
A function can appear in several forms:

- a table
- a graph
- an equation
- a set of ordered pairs

The rule does not change just because the representation changes.

For example, these could all describe the same function:

- table values that increase by `4`
- points on a line
- the equation `y = 4x + 1`

The main test is always the same:

Does each input match exactly one output?

If yes, the relationship is a function.

## Worked Example 1: Writing a Rule From a Table
Study the table:

| x | y |
| --- | --- |
| 0 | 2 |
| 1 | 5 |
| 2 | 8 |
| 3 | 11 |

Step 1: Look for the pattern in the outputs.

The outputs increase by `3` each time the input increases by `1`.

So the rate of change is `3`.

Step 2: Identify the initial value.

When `x = 0`, the output is `2`.

So the initial value is `2`.

Step 3: Write the equation.

`y = 3x + 2`

Check with `x = 3`:

`y = 3(3) + 2 = 11`

The rule matches the table.

## Worked Example 2: Deciding Whether a Graph Represents a Function
Suppose a graph includes the points:

- `(1, 2)`
- `(2, 4)`
- `(3, 6)`
- `(3, 8)`

This graph does not represent a function.

Why?

The input `3` is paired with two different outputs: `6` and `8`.

That means a vertical line at `x = 3` would hit the graph at two points.

This leads to a useful test:

- if any vertical line crosses the graph more than once, the graph is not a function

This is called the vertical line test.

## Worked Example 3: Comparing Two Functions
Function A is shown by the equation:

`y = 4x + 1`

Function B is shown by the table:

| x | y |
| --- | --- |
| 0 | 3 |
| 1 | 5 |
| 2 | 7 |

Compare their rates of change and initial values.

Function A:

- rate of change: `4`
- initial value: `1`

Function B:

- outputs increase by `2`, so the rate of change is `2`
- when `x = 0`, the output is `3`, so the initial value is `3`

Interpretation:

- Function A grows faster because `4 > 2`
- Function B starts higher because `3 > 1`

A strong comparison does not only say which function is "bigger." It explains whether you are comparing the starting point or the rate of change.

## Worked Example 4: Using a Function in Context
A gym charges a sign-up fee of `$15` and then `$6` for each class.

Let `x` be the number of classes and `y` be the total cost.

Step 1: Write the function rule.

`y = 6x + 15`

Step 2: Interpret the numbers.

- `6` is the rate of change, or cost per class
- `15` is the initial value, or starting fee

Step 3: Find the cost for `8` classes.

`y = 6(8) + 15`

`y = 48 + 15`

`y = 63`

So the total cost is `$63`.

## Equation, Table, and Graph Tell the Same Story
Consider the function:

`y = 2x + 5`

You can understand it in several ways:

- the equation shows the rule directly
- the table shows several matching pairs
- the graph shows the steady increase visually

If you make a table, you get:

| x | y |
| --- | --- |
| 0 | 5 |
| 1 | 7 |
| 2 | 9 |

That means the graph would pass through `(0, 5)`, `(1, 7)`, and `(2, 9)`.

Being able to move between these forms is one of the main goals of introductory function work.

## Common Mistake: Mixing Up Initial Value and Rate of Change
Students sometimes see the equation:

`y = 5x + 2`

and say:

- the rate of change is `2`
- the initial value is `5`

That is backward.

In `y = mx + b`:

- `m` is the rate of change
- `b` is the initial value

So for `y = 5x + 2`:

- rate of change = `5`
- initial value = `2`

You can check by substituting `x = 0`. The result is `2`, which confirms the initial value.

## Summary
A function matches each input with exactly one output.

To analyze introductory functions well:

1. test whether the relationship follows the one-input, one-output rule
2. find the rate of change by checking how outputs increase
3. find the initial value by looking at the output when `x = 0`
4. compare functions by discussing both how they start and how they grow

When you connect tables, graphs, equations, and context, functions become a powerful way to describe changing quantities.
