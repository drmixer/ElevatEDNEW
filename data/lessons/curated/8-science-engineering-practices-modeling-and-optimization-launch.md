# Modeling & Optimization

## Learning Goal
- I can explain how models help engineers test ideas before building full solutions.
- I can use model results to identify ways to optimize a design.
- I can support a claim about design improvement using evidence from repeated tests.

## Introduction
Engineers rarely jump straight from an idea to a finished design. They use models to predict performance, explore variables, and find weak points before investing more time or materials.

Optimization means improving a design step by step based on evidence. In this lesson, ask:

1. How does a model help engineers reason about a design?
2. What variable should be changed when the first design underperforms?
3. How do repeated tests support optimization?

## Key Vocabulary
**Model**: A representation of a system or design used to explain, predict, or test behavior.

**Optimization**: Improving a design so it better meets the goal.

**Variable**: A factor that can be changed in a test or design.

**Prototype**: An early version of a design built for testing.

**Efficiency**: How well a system achieves a result with limited waste or cost.

**Performance**: How well a design meets its criteria.

## Phenomenon: A Water Filter Does Not Clean Fast Enough
A class designs a small water filter for muddy water. Their first prototype removes most visible particles, but the flow is very slow. Students compare several filter models and record:

- how clear the water looks after filtering
- how long it takes one cup of water to pass through
- which material layers were used
- whether the filter leaks or collapses

The question is:

How can models and repeated tests help engineers improve both filtering quality and speed?

## Why Engineers Use Models
A model can be physical, diagram-based, mathematical, or digital. Its job is not to be perfect. Its job is to help engineers reason about the system.

Models can help engineers:

- visualize how parts interact
- predict which variables matter most
- test one change at a time
- avoid wasting materials on full-scale builds too early

This matters because complex designs are easier to improve when the system is broken into variables engineers can compare.

## Optimization Requires Repeated Testing
Optimization is not random trial and error. Engineers use evidence from one version to improve the next version.

For the filter example, useful variables might include:

- thickness of the gravel or sand layers
- order of the materials
- size of the outlet opening
- shape of the filter container

The engineering goal is to change the design in a way that improves performance without creating a bigger problem somewhere else.

## Worked Example 1: Using a Model to Predict a Change
Suppose students sketch a filter model showing that water moves through three very dense layers before reaching the outlet.

What is a reasonable prediction?

"The design may filter particles well, but it may also slow water flow too much because the path creates a lot of resistance."

This is a strong model-based claim because it links the structure of the system to a predicted outcome before the next build.

## Worked Example 2: Optimizing With Evidence
Prototype A:

- produces clear water
- takes 180 seconds per cup

Prototype B:

- produces slightly less clear water
- takes 70 seconds per cup

Prototype C:

- produces clear water similar to A
- takes 95 seconds per cup

Which design looks most optimized so far?

A strong answer is:

"Prototype C appears most optimized because it keeps the high water clarity while improving speed much more than Prototype A."

Reasoning:

- optimization means balancing the important performance criteria
- Prototype C preserves the main benefit while improving a key weakness

## Worked Example 3: One Variable at a Time
Why do engineers often change only one variable between tests?

A strong explanation is:

"Changing one variable at a time makes it easier to identify which change caused the performance difference."

If students changed the container shape, the material order, and the outlet size all at once, they would have difficulty knowing which revision improved the result.

## Common Mistake: Calling the Fastest Design the Best
Students sometimes say:

"The fastest filter wins."

That ignores the full goal.

A stronger method is:

1. define the important performance measures
2. compare designs across all of them
3. use the evidence to decide which version is actually more optimized

## Summary
Modeling and optimization help engineers improve designs through evidence-based revision.

Key ideas from this lesson:

1. models help engineers predict system behavior and choose what to test
2. optimization uses repeated evidence to improve performance
3. strong engineering decisions compare several criteria, not just one convenient result

When you study an engineering design, ask what the model predicts, what the tests reveal, and how the next revision should improve the design.
